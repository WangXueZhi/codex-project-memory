# 架构说明

[English](architecture.md) | 简体中文

## 组件

本仓库是一个 Codex marketplace，包含一个自包含插件。插件打包精简 Skill、本地 CLI 和
Stop Hook。CLI 负责确定性存储和访问校验，Skill 负责交互审核或自动回退策略，不需要
MCP 服务器。

## 项目标识

注册从规范化文件系统路径开始。Git 仓库还会记录 worktree 公共目录、origin URL 和当前
提交，作为项目移动后的识别线索。相似的新路径只会作为候选返回，未经确认不会自动合并。

## 存储

所有状态位于 `CODEX_HOME/project-memory/v1`，不写入已注册项目。全局 `registry.json`
记录项目位置，`links.json` 记录单向读取授权。每个项目目录包含 `project.json`、审核通过的
`MEMORY.md`、proposal JSON、`RELATIONS.json` 和追加式 `audit.jsonl`。

记忆文档 v2 支持摘要、主题和多个已校验来源；v1 仍可读取，只有在记忆或补全实际提交时才
升级。目录使用 `0700`，状态文件使用 `0600`，写入采用私有临时文件加原子重命名。

proposal 和正式记忆严格分离。结构化输入可用时，用户选择提交或拒绝；不可用时，Skill
执行明确标记的全部提交回退。搜索直接读取 Markdown，不维护数据库索引。

## 知识图谱

每条审核通过的记忆 UUID 是一个节点。类型化边保存在项目的 `RELATIONS.json` 中；缺少该
文件视为空图。关系类型包括 `related_to`、`depends_on`、`supports`、`contradicts`、
`supersedes` 和 `derived_from`。其中相关和矛盾是双向关系，其余为有向关系。

关系候选与记忆 proposal 共用审核生命周期，可引用已有记忆 ID 或同一 proposal 中的稳定
candidate ref。

普通加载和文本搜索不受图谱影响。只读 `guide` 会计算主题、正式连通分量、孤立记忆、证据
覆盖、失效来源、重点记忆、建议问题和确定性关联线索。线索仅根据已审核元数据中的共享来源、
相同主题和稀有共享标签生成；公共来源、公共标签和已有正式边会被排除。线索不会写入、计入
正式关系或参与遍历，除非用户明确选择并通过原有审核流程。

邻居、最短路径和有界图查询均需显式执行，可输出 JSON、Mermaid、Markdown 或私有离线
HTML。HTML 使用构建时打包的 Preact 和 Cytoscape；FCoSE 用于力导向图，Dagre 用于
层级阅读布局。JavaScript 和 CSS 以内联形式写入，并通过 CSP 哈希限制。页面不使用 CDN、
服务器或运行时网络请求。

浏览器默认显示知识导览，再提供深色图谱和阅读详情。图谱节点按主题着色，正式关系采用类型化
线条，待审核线索使用低亮度虚线。动画可暂停，并遵循 reduced-motion 设置。

来源文件不是正式关系。HTML 只在用户展开当前记忆时派生证据、报告、流程和参考节点；这些
节点不会进入关系计数、遍历、存储或导出数据。

跨项目边归当前项目所有，至少包含一个本地端点，并要求当前单向项目链接持续授权所有外部
端点。删除项目链接后边会暂停显示，但不会被删除。

## 任务结束检查

Stop Hook 在已注册项目的工具任务结束时运行。pending proposal 不是有效结束状态。Hook
要求交互式提交/拒绝回执、自动保存回执或明确的无更新回执。它使用与 Skill CLI 相同的文件
存储代码，不依赖 MCP。

## 跨项目只读

项目 B 到项目 A 的链接只允许 B 搜索 A 的记忆和读取允许的文本文件，不包含反向权限。所有
文件请求都会规范化路径、限制在目标根目录内、检查内置和用户拒绝规则，并拒绝符号链接逃逸。

## 过期检查

每条记忆最多可引用十二个当前项目或已授权项目的来源。插件保存来源提交版本和 SHA-256
哈希，读取时逐一重新计算；变化、不可用或失去授权的来源会分别报告。

## 非目标

本项目不提供云同步、团队共享记忆、静态加密、图形化编辑管理、embeddings、自动关系激活、
自动 Git 修改或后台记忆抽取。
