# Support

English | [简体中文](SUPPORT.zh-CN.md)

## Get Help

Use [GitHub Issues](https://github.com/WangXueZhi/codex-project-memory/issues) for installation
problems, bug reports, documentation questions, and feature requests.

Before opening an issue:

1. Check that you are using Node.js 22.13 or newer.
2. Run `pnpm check` when reporting a source-build problem.
3. Search existing issues for the same error.
4. Remove credentials, private source code, project paths, memory state, and personal information.

Include the plugin version, operating system, Codex surface, exact command, expected result, actual
result, and the smallest safe reproduction you can provide.

## Security Reports

Do not report vulnerabilities or sensitive data publicly. Use the
[private security advisory page](https://github.com/WangXueZhi/codex-project-memory/security/advisories/new)
and follow [SECURITY.md](SECURITY.md).

## Support Scope

Community support is provided on a best-effort basis. There is no guaranteed response time or
service-level agreement. Support does not include recovering deleted local memory, accessing private
projects, managing Codex or OpenAI accounts, or troubleshooting unsupported forks.
