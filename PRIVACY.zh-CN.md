# 隐私政策

[English](PRIVACY.md) | 简体中文

生效日期：2026 年 7 月 15 日

本政策说明 Codex Project Memory 插件怎样处理数据。它只适用于插件本身，不适用于 Codex、
ChatGPT、GitHub、操作系统或用户选择配合使用的其他服务。

## 插件会处理什么数据

插件可能处理：

- 用户明确注册的项目路径和 Git 信息。
- 记忆候选、正式记忆、摘要、主题、标签和关系。
- 本地证据、报告、流程和参考文件的位置。
- 用于发现来源变化的文件哈希和 Git 提交标识。
- 记录记忆操作的本地审计信息。

插件不需要用户的姓名、电子邮箱、付款信息或在线账号。

## 数据保存在哪里

插件数据保存在本机：

```text
${CODEX_HOME:-~/.codex}/project-memory/v1/
```

用户可以通过 `CODEX_PROJECT_MEMORY_HOME` 选择其他本地目录。记忆文件不会写入已注册项目
目录。

## 网络和数据共享

插件没有运行时网络客户端、云端服务、遥测、广告或分析功能。它不会把记忆内容、文件内容、
项目路径或使用信息发送给开发者。

Codex 或其他宿主程序可能按照自己的隐私条款处理用户提示和工具操作。这部分处理不属于本
插件，也不受本政策控制。

## 文件访问

插件只读取用户明确注册的项目。当前项目要读取另一个已注册项目，必须先建立明确的单向链接。
跨项目访问始终为只读。

内置拒绝规则会阻止常见凭据和密钥文件，用户还可以通过本地 `denyPatterns` 禁止更多路径。
这些检查只能降低风险，用户仍应把密钥放在源码目录之外，并认真检查项目授权。

## 保存行为

可以进行结构化审核时，插件会询问用户“保存全部、选择保存或暂不保存”。无法进行这种交互时，
当前回退策略会保存全部候选项，并在最终回复中明确说明自动操作。

请勿把密钥、凭据、个人数据或不应成为长期本地记忆的机密内容提交为记忆候选。

## 保留和删除

数据会一直保存在用户电脑上，直到用户删除。用户可以：

- 使用 `forget` 删除指定记忆。
- 使用 `forget-relations` 删除指定关系。
- 删除配置的项目记忆数据目录，移除全部插件数据。

如果以后可能需要恢复，请在删除前备份该目录。

## 安全

操作系统支持时，插件会使用私人文件权限。插件还会检查路径、拒绝符号链接逃逸、限制文件读取
量，并通过一次性替换写入状态文件。安全问题请通过项目的
[私有安全公告页面](https://github.com/WangXueZhi/codex-project-memory/security/advisories/new)报告。

## 政策变更

重要变更会记录在仓库中，并同步更新生效日期。

## 联系方式

隐私问题可以在不包含敏感信息的前提下提交到
[GitHub Issues](https://github.com/WangXueZhi/codex-project-memory/issues)。包含敏感细节的
报告请使用私有安全公告。
