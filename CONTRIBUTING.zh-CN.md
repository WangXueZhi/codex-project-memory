# 贡献指南

[English](CONTRIBUTING.md) | 简体中文

开发环境需要 Node.js 22.13 或更高版本，以及 pnpm 10.30.2。

准备依赖并运行完整本地检查：

```bash
pnpm install
pnpm check
```

`pnpm check` 会依次执行类型检查、代码质量检查、单元测试、正式构建、集成测试和项目验证。
修改离线知识浏览器时，还要运行 `pnpm test:visual`。

涉及文件读写的修改需要测试：被禁止的路径、试图离开项目目录的路径、符号链接、私人权限、
写入中断，以及单向项目链接。如果修改了存储文件格式，必须说明老用户如何继续读取旧数据，
或者提供清楚的迁移方法。

未经单独设计评审，请勿加入遥测、运行时网络访问、向已注册项目目录写入文件或跨项目写入。
修改保存策略时，还必须说明用户是否会逐条审核，以及交互不可用时是否存在自动提交回退。
