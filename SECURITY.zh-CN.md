# 安全策略

[English](SECURITY.md) | 简体中文

## 信任模型

Codex Project Memory 是一个本地进程，只访问用户明确注册的路径。跨项目访问需要单向授权
链接；插件不提供跨项目写入，也不访问网络。

插件会阻止常见敏感路径和文件类型，拒绝父目录穿越和符号链接逃逸，限制文件大小和搜索输出，
并拒绝在记忆候选中保存疑似凭据。插件状态存放在项目目录之外，目录和文件使用私有权限，并
通过原子替换写入。这些措施不能替代将密钥保存在源码目录之外的基本安全实践。

当 Codex 提供 `request_user_input` 时，Skill 会询问“保存全部 / 选择保存 / 暂不保存”。
该工具不可用时，当前策略会自动提交所有候选记忆和关系，并在最终回复中标明。必须强制人工
审核的用户应先调整 Skill 策略，再启用插件。

## 报告漏洞

请勿在公开安全报告中包含真实凭据、私有源码或项目记忆状态文件。请通过 GitHub 私有安全
公告提交漏洞：

https://github.com/WangXueZhi/codex-project-memory/security/advisories/new

## 删除数据

使用 `memory_forget` 删除指定记忆。若需删除全部插件数据，请先停止 Codex，再删除配置的
项目记忆数据目录。需要恢复时，请在删除前备份。
