# 安全策略

[English](SECURITY.md) | 简体中文

## 信任模型

简单来说，安装插件后，它可以读取用户明确注册的项目目录。另一个项目即使已经注册，也不会
因此自动开放读取；用户必须先批准单向链接。插件不提供跨项目写入，也不访问网络。

插件会阻止常见密钥文件，拒绝试图离开已授权项目目录的路径，限制单次读取的数据量，并拒绝
在记忆候选中保存疑似凭据。插件状态存放在项目目录之外；操作系统支持时，私人文件权限只
允许当前用户访问。更新文件时采用一次性替换，减少写入中断造成存储损坏的风险。这些措施
可以降低风险，但真实密钥仍然应该放在源码目录之外。

当 Codex 提供 `request_user_input` 时，Skill 会询问“保存全部 / 选择保存 / 暂不保存”。
该工具不可用时，当前策略会自动提交所有候选记忆和关系，并在最终回复中标明。必须强制人工
审核的用户应先调整 Skill 策略，再启用插件。

## 报告漏洞

请勿在公开安全报告中包含真实凭据、私有源码或项目记忆状态文件。请通过 GitHub 私有安全
公告提交漏洞：

https://github.com/WangXueZhi/codex-project-memory/security/advisories/new

## 删除数据

使用 `forget` 删除指定记忆，使用 `forget-relations` 删除指定关系。若需删除全部插件数据，
请先停止 Codex，再删除配置的项目记忆数据目录。需要恢复时，请在删除前备份。
